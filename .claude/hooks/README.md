This workspace ships a `.claude/` hook fabric: small bash gates wired into Claude
Code lifecycle events (`PreToolUse`, `PostToolUse`, `UserPromptSubmit`,
`SessionStart`) via `.claude/settings.json`. Each hook sources
`.claude/hooks/lib/hooklib.sh` for shared mechanics (input parsing, the disable
contract, audit plumbing, secret matching). Every row below is derived from
settings.json, hooklib, and the hook source — never hand-maintained.

**Columns.**
- **class** — `block` exits non-zero to stop the tool call; `inject` adds
  advisory context (`additionalContext` or stdout) without blocking; `warn`
  prints an advisory to stdout/stderr; `-` is a pure side-effect recorder.
- **pinned?** — `yes` hooks (`HOOK_PINNED`) can never be disabled; security /
  non-negotiable enforcement.
- **disable id** — token to add to `WORKFLOW_DISABLED_HOOKS` (comma-separated) to
  turn a non-pinned hook off; `WORKFLOW_HOOKS_OFF=1` disables all non-pinned hooks.
- **failure mode** — `fail-closed` (`HOOK_FAILCLOSED`) blocks the tool call if its
  `jq` dependency is broken (better no mutation than an unscanned one); `fail-open`
  degrades to a no-op with one stderr line.
- **test file** — the hook's smoke test under `.claude/hooks/tests/`, run by
  `make hooks-test`.

## Boundary registrations (not hook scripts)

Two lifecycle registrations in `settings.json` are deliberately **not**
`.claude/hooks/*.sh` scripts and so are not table rows above:

- **`scripts/check-symlinks.sh`** — a `SessionStart` registration that lives under
  `scripts/` (a workspace tool, not a fabric hook). It validates the workspace
  symlink map and is appended with `|| true` so it never blocks a session.
- **`.claude/hooks/statusline.js`** — the `statusLine` command (Node, not a bash
  hook). It renders the status line and is not part of the gate fabric.

## Adopting the fabric

To vendor this hook fabric into another repo:

1. **Copy the hook directory.** Vendor `.claude/hooks/` (the `*.sh` hooks, the
   `lib/hooklib.sh` shared library, and `tests/`) into the target repo.
2. **Wire `settings.json`.** Register the hooks under the matching lifecycle
   events (`PreToolUse`/`PostToolUse`/`UserPromptSubmit`/`SessionStart`) with the
   `matcher` values shown above. Use `${CLAUDE_PROJECT_DIR:-.}` in each command so
   paths resolve regardless of cwd.
3. **Install `jq`.** Every hook inspects its JSON input with `jq`. Without it,
   `fail-open` hooks degrade to no-ops (one stderr line) and `fail-closed`
   (`HOOK_FAILCLOSED`) hooks block — so `jq` is a hard dependency for the
   security gates.
4. **Know the escape hatches.** Set them in the launching environment
   (`settings.json` `env`), not per-command:
   - `WORKFLOW_DISABLED_HOOKS=<id>,<id>` — disable specific non-pinned hooks by
     their **disable id** (the table column).
   - `WORKFLOW_HOOKS_OFF=1` — disable all non-pinned hooks at once.
   - **Pinned** hooks (`HOOK_PINNED`, the `pinned? = yes` rows) ignore both — they
     are security non-negotiables and stay on.
5. **Understand the failure contract.** `fail-open` is the default: a broken
   dependency degrades to a no-op. The `fail-closed` set (`HOOK_FAILCLOSED`)
   instead blocks the tool call, because letting an unscanned mutation through is
   worse than blocking it. The two sets are orthogonal to the pinned set.
6. **Smoke-test and inspect.** Run `make hooks-test` (shellcheck + each hook's
   `tests/test-<id>.sh`) to verify the fabric loads. Run `make wf-status` (or
   `bash .claude/hooks/lib/hooklib.sh status`) to print which hooks are
   enabled/disabled in the current environment.
